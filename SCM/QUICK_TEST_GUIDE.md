# 快速測試指南

這是一份簡化版的測試指南，幫助你快速驗證系統功能。

## 🚀 快速開始

1. **啟動系統**
   ```bash
   npm run dev
   ```
   訪問：`http://localhost:3000/scm`

2. **從首頁開始**
   - 確認看到三個模組卡片
   - 依序點擊每個模組進行測試

---

## 📋 5 分鐘快速測試

### ✅ 測試 1：EDA 模組（2 分鐘）
1. 點擊「M1: 探索性數據分析」
2. **檢查：**
   - [ ] 看到表格，有數據顯示
   - [ ] 點擊表格行，上半部顯示圖表
   - [ ] 點擊藍色連結（如 "Alcohol"），進入詳細視圖
   - [ ] 在詳細視圖點擊「關聯事件」按鈕，看到彈窗
   - [ ] 點擊「返回總覽」回到列表

### ✅ 測試 2：預測分析模組（2 分鐘）
1. 從首頁點擊「M2: 預測分析駕駛艙」
2. **檢查：**
   - [ ] 看到「違規告警」表格
   - [ ] 點擊「查看詳情」，看到圖表（粉紅線+紅虛線）
   - [ ] 切換到「算法洞察」Tab，看到算法統計表格
   - [ ] 切換到「算法設定」Tab，可以啟用/停用算法

### ✅ 測試 3：需求假設模組（1 分鐘）
1. 從首頁點擊「M3: 需求假設與預測層」
2. **檢查：**
   - [ ] 看到 Summary 表格，點擊一行
   - [ ] 下半部顯示 Details 表格（時間欄位表頭是黃色）
   - [ ] 點擊數值可以編輯
   - [ ] 切換到「預測圖層」Tab，看到瀑布圖

---

## 🎨 視覺檢查（30 秒）

快速檢查配色是否符合規格：

- [ ] 背景是深灰色（`#1F1F1F`）
- [ ] 文字是淺灰色（`#E0E0E0`）
- [ ] 連結是藍色（`#3B82F6`）
- [ ] 選中的表格行是亮藍色背景
- [ ] 時間欄位表頭是黃色（`#FCD34D`）

---

## ⚠️ 如果遇到問題

1. **頁面無法載入**
   - 檢查 `npm run dev` 是否正常運行
   - 檢查瀏覽器控制台是否有錯誤

2. **樣式不對**
   - 確認 `tailwind.config.js` 已包含 SCM 路徑
   - 重新啟動開發伺服器

3. **功能不工作**
   - 檢查瀏覽器控制台的錯誤訊息
   - 確認所有檔案都已正確建立
