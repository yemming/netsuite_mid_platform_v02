# NetSuite Platform V2 - 設計系統文件

> **版本**: 1.0.0  
> **最後更新**: 2025-01-XX  
> **用途**: 定義整個 NetSuite Platform V2 的視覺設計規範和元件標準

---

## 📋 設計系統概述

本設計系統基於 **NetSuite Next UI** 的視覺風格，提供一套完整、一致、可擴展的設計規範。所有頁面、元件和互動都應遵循此設計系統，確保整個平台的視覺一致性和使用者體驗的統一性。

### 設計原則

1. **一致性 (Consistency)**: 所有元件使用統一的設計語言
2. **可讀性 (Readability)**: 優化文字大小、行高和對比度
3. **可訪問性 (Accessibility)**: 符合 WCAG 2.1 AA 標準
4. **響應式 (Responsive)**: 適配各種螢幕尺寸
5. **效能 (Performance)**: 輕量級設計，快速載入

## 🎨 色彩系統

### 主色調 (Primary Colors)

NetSuite 採用藍綠色作為品牌主色，代表專業、可靠和創新。

#### 主色 (Primary)
- **色相**: 203° (藍綠色)
- **飽和度**: 60%
- **明度**: 45%
- **HSL**: `hsl(203, 60%, 45%)`
- **用途**: 主要按鈕、連結、焦點狀態、重要元素

#### 主色變體
- **Primary Light**: `hsl(203, 60%, 55%)` - 懸停狀態
- **Primary Dark**: `hsl(203, 60%, 35%)` - 按下狀態
- **Primary Background**: `hsl(203, 60%, 95%)` - 淺色背景

### 語義色彩 (Semantic Colors)

#### 成功 (Success)
- **主色**: `hsl(142, 71%, 45%)` - 綠色
- **用途**: 成功訊息、完成狀態、正向操作

#### 警告 (Warning)
- **主色**: `hsl(38, 92%, 50%)` - 橙色
- **用途**: 警告訊息、注意事項、待處理狀態

#### 錯誤 (Error / Destructive)
- **主色**: `hsl(0, 84%, 60%)` - 紅色
- **用途**: 錯誤訊息、刪除操作、危險操作

#### 資訊 (Info)
- **主色**: `hsl(203, 60%, 45%)` - 與主色相同
- **用途**: 資訊提示、說明文字

### 中性色 (Neutral Colors)

#### 淺色模式 (Light Mode)
- **背景**: `hsl(0, 0%, 100%)` - 純白
- **前景**: `hsl(0, 0%, 13%)` - 深灰（主要文字）
- **次要文字**: `hsl(0, 0%, 40%)` - 中灰
- **邊框**: `hsl(0, 0%, 91%)` - 淺灰
- **次要背景**: `hsl(0, 0%, 96%)` - 極淺灰

#### 深色模式 (Dark Mode)
- **背景**: `hsl(203, 22%, 20%)` - NetSuite 深藍綠 (#28363F)
- **前景**: `hsl(0, 0%, 100%)` - 純白
- **卡片背景**: `hsl(203, 22%, 23%)` - 稍亮的深藍綠
- **次要背景**: `hsl(203, 18%, 28%)` - 中深藍綠
- **次要文字**: `hsl(0, 0%, 70%)` - 淺灰
- **邊框**: `hsl(203, 18%, 28%)` - 與次要背景相同

### 色彩使用指南

1. **主色使用**: 僅用於主要操作和重要元素，避免過度使用
2. **對比度**: 文字與背景的對比度至少 4.5:1（AA 標準）
3. **深色模式**: 所有色彩都應提供深色模式版本
4. **語義色彩**: 僅用於對應的語義情境，不要混用

## 📝 字體系統

### 字體家族 (Font Family)

- **無襯線字體**: `system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`
- **等寬字體**: `'Courier New', Courier, monospace` (用於程式碼)

### 字體大小階層 (Font Size Scale)

| 名稱 | 大小 | 行高 | 用途 |
|------|------|------|------|
| `xs` | 12px (0.75rem) | 18px (1.125rem) | 小字、描述文字、標籤 |
| `sm` | 14px (0.875rem) | 20px (1.25rem) | 中文字、用戶名、次要文字 |
| `base` | 14px (0.875rem) | 19.6px (1.4rem) | 基礎字體、正文 |
| `lg` | 16px (1rem) | 24px (1.5rem) | 較大正文、次要標題 |
| `xl` | 18px (1.125rem) | 31.5px (1.75rem) | 小標題 |
| `2xl` | 20px (1.25rem) | 30px (1.875rem) | 中標題 |
| `3xl` | 24px (1.5rem) | 32px (2rem) | 大標題 |
| `4xl` | 30px (1.875rem) | 36px (2.25rem) | 超大標題 |

### 字重 (Font Weight)

- **Light (300)**: 極少使用
- **Regular (400)**: 預設字重，用於正文
- **Medium (500)**: 次要標題、強調文字
- **Semibold (600)**: 重要標題、按鈕文字
- **Bold (700)**: 主要標題、重要強調

### 行高 (Line Height)

- **緊湊**: 1.125 - 1.25 (標題、單行文字)
- **標準**: 1.4 - 1.5 (正文、多行文字)
- **寬鬆**: 1.6 - 1.75 (長篇文章、說明文字)

### 字體使用指南

1. **基礎字體**: 所有頁面基礎字體大小為 14px
2. **標題階層**: 使用明確的標題階層 (h1-h6)
3. **行高**: 中文內容建議使用 1.6 的行高
4. **字重**: 避免使用過多字重，保持簡潔

## 📏 間距系統 (Spacing System)

### 基礎單位

- **基礎單位**: 4px
- **命名**: 使用 Tailwind CSS 的間距系統

### 間距階層

| Tailwind 類別 | 實際大小 | 用途 |
|--------------|----------|------|
| `0` | 0px | 無間距 |
| `1` | 4px | 極小間距（圖示與文字） |
| `2` | 8px | 小間距（表單元素間） |
| `3` | 12px | 中小間距 |
| `4` | 16px | 標準間距（元件間距） |
| `6` | 24px | 中等間距（區塊間距） |
| `8` | 32px | 大間距（區塊間距） |
| `12` | 48px | 超大間距（頁面區塊） |
| `16` | 64px | 極大間距（主要區塊） |

### 間距使用指南

1. **一致性**: 使用預定義的間距值，避免自訂數值
2. **視覺節奏**: 相關元素使用較小間距，無關元素使用較大間距
3. **響應式**: 在行動裝置上可適當縮小間距

## 🔲 圓角系統 (Border Radius)

### 圓角大小

- **無圓角**: `0` - 用於全寬元素
- **小圓角**: `0.25rem` (4px) - 用於小按鈕、標籤
- **中圓角**: `0.375rem` (6px) - **NetSuite 預設**，用於卡片、按鈕
- **大圓角**: `0.5rem` (8px) - 用於大型卡片、模態框

### 圓角使用指南

1. **預設值**: 使用 `0.375rem` 作為預設圓角
2. **一致性**: 相同類型的元件使用相同的圓角大小
3. **避免過度**: 不要使用過大的圓角（> 12px）

---

## 🎭 陰影系統 (Shadow System)

### 陰影階層

| 名稱 | 陰影值 | 用途 |
|------|--------|------|
| `sm` | `0 1px 2px 0 rgba(0, 0, 0, 0.05)` | 輕微提升（輸入框） |
| `default` | `0 1px 3px 0 rgba(0, 0, 0, 0.1), 0 1px 2px 0 rgba(0, 0, 0, 0.06)` | 標準提升（卡片） |
| `md` | `0 4px 6px -1px rgba(0, 0, 0, 0.1), 0 2px 4px -1px rgba(0, 0, 0, 0.06)` | 中等提升（懸停卡片） |
| `lg` | `0 10px 15px -3px rgba(0, 0, 0, 0.1), 0 4px 6px -2px rgba(0, 0, 0, 0.05)` | 大提升（模態框） |
| `xl` | `0 20px 25px -5px rgba(0, 0, 0, 0.1), 0 10px 10px -5px rgba(0, 0, 0, 0.04)` | 超大提升（下拉選單） |

### 深色模式陰影

深色模式下，陰影應使用較深的顏色和較高的不透明度：
- 使用 `rgba(0, 0, 0, 0.3)` 或更高的不透明度
- 或使用 `rgba(255, 255, 255, 0.1)` 作為亮色陰影

## 🧩 元件規範

### 按鈕 (Button)

#### 尺寸
- **小 (sm)**: 高度 36px (2.25rem)，內邊距 12px (0.75rem)
- **中 (default)**: 高度 40px (2.5rem)，內邊距 16px (1rem)
- **大 (lg)**: 高度 44px (2.75rem)，內邊距 32px (2rem)
- **圖示 (icon)**: 40x40px 正方形

#### 變體
- **Primary**: 主色背景，白色文字
- **Secondary**: 淺灰背景，深色文字
- **Outline**: 透明背景，主色邊框和文字
- **Ghost**: 透明背景，懸停時顯示背景
- **Destructive**: 紅色背景，白色文字

#### 狀態
- **Default**: 正常狀態
- **Hover**: 背景色加深 10%
- **Active**: 背景色加深 20%
- **Disabled**: 50% 不透明度，無法點擊

### 輸入框 (Input)

#### 尺寸
- **高度**: 40px (2.5rem)
- **內邊距**: 12px 水平，10px 垂直
- **字體大小**: 14px

#### 樣式
- **邊框**: 1px 實線，顏色 `hsl(0, 0%, 91%)`
- **圓角**: 6px (0.375rem)
- **焦點**: 2px 主色邊框，無陰影

#### 狀態
- **Default**: 淺灰邊框
- **Focus**: 主色邊框
- **Error**: 紅色邊框
- **Disabled**: 淺灰背景，50% 不透明度

### 卡片 (Card)

#### 結構
- **內邊距**: 24px (1.5rem)
- **邊框**: 1px 實線，顏色 `hsl(0, 0%, 91%)`
- **圓角**: 6px (0.375rem)
- **陰影**: 標準陰影 (default)

#### 深色模式
- **背景**: `hsl(203, 22%, 23%)`
- **邊框**: `hsl(203, 18%, 28%)`

### 表格 (Table)

#### 結構
- **表頭**: 淺灰背景，粗體文字
- **行高**: 48px (3rem)
- **內邊距**: 16px 水平，12px 垂直
- **分隔線**: 1px 實線，顏色 `hsl(0, 0%, 91%)`

#### 樣式
- **斑馬紋**: 可選，使用極淺灰背景
- **懸停**: 淺灰背景高亮
- **選中**: 主色背景，白色文字

## 📱 響應式設計

### 斷點 (Breakpoints)

| 名稱 | 最小寬度 | 用途 |
|------|----------|------|
| `sm` | 640px | 小型平板 |
| `md` | 768px | 平板 |
| `lg` | 1024px | 筆記型電腦 |
| `xl` | 1280px | 桌面 |
| `2xl` | 1536px | 大型桌面 |

### 響應式策略

1. **行動優先**: 從行動裝置開始設計
2. **彈性佈局**: 使用 Grid 和 Flexbox
3. **斷點使用**: 僅在必要時使用斷點
4. **觸控友好**: 按鈕和連結至少 44x44px

## 🌓 深色模式

### 切換機制

- 使用 `ThemeProvider` 元件管理主題
- 支援系統偏好設定自動切換
- 提供手動切換開關

### 色彩適配

所有色彩都應提供深色模式版本：
- 使用 CSS 變數 (`--variable`) 管理
- 透過 `.dark` 類別切換
- 確保對比度符合可訪問性標準

## 🎯 使用指南

### 開發流程

1. **選擇元件**: 從設計系統中選擇合適的元件
2. **套用樣式**: 使用預定義的 Tailwind 類別
3. **保持一致**: 遵循既定的設計規範
4. **測試響應式**: 確保在不同裝置上正常顯示
5. **測試深色模式**: 確保深色模式下可讀性良好

### 自訂元件

如果需要建立自訂元件：
1. 參考現有元件的設計模式
2. 使用設計系統中的色彩、間距、字體
3. 確保符合可訪問性標準
4. 提供深色模式支援

### 主題切換

要切換到不同的設計主題：
1. 修改 `config/design-tokens/` 中的 JSON 配置檔
2. 執行主題生成腳本（如果有的話）
3. 重新建置應用程式
